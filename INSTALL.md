# Инструкция по развёртыванию GophKeeper на чистой Ubuntu с использованием Docker

## Часть 1. Введение и подготовка системы

Данное руководство описывает полную установку, настройку и запуск клиент-серверной системы хранения секретов GophKeeper на Ubuntu с использованием **Docker** для серверной части и сборки клиентского бинарника из исходного кода. Инструкция разбита на 5 логических частей для удобства восприятия.

### 1.1. Обновление системы и установка базовых пакетов

Начните с обновления репозиториев и установки необходимых инструментов:

```bash
sudo apt update
sudo apt upgrade -y
sudo apt install -y git make wget curl gcc build-essential openssl
```

### 1.2. Установка Docker и Docker Compose

Серверная часть работает в контейнерах, поэтому установите Docker и плагин Compose:

```bash
# Добавляем официальный GPG-ключ Docker
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Добавляем репозиторий
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Устанавливаем Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Добавляем пользователя в группу docker (чтобы избежать sudo)
sudo usermod -aG docker $USER
newgrp docker
```

Проверьте установку:

```bash
docker --version
docker compose version
```

### 1.3. Установка Go (необходим для сборки клиента)

Клиент собирается из исходного кода, поэтому требуется Go 1.26 или новее:

```bash
wget https://go.dev/dl/go1.26.0.linux-amd64.tar.gz
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go1.26.0.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
echo 'export GOPATH=$HOME/go' >> ~/.bashrc
source ~/.bashrc
go version
```

## Часть 2. Получение исходного кода, генерация сертификатов и настройка SSH-агента

### 2.1. Клонирование репозитория

```bash
git clone https://github.com/IlVin/GophKeeper.git
cd GophKeeper
```

Загрузите все зависимости Go:

```bash
go mod download
```

### 2.2. Генерация PKI-сертификатов (требуется для mTLS)

Сервер использует собственные центры сертификации. Выполните команду:

```bash
make certs
```

Эта команда создаст:

Приватные ключи в папке .certs_private/ (эти файлы монтируются в контейнер сервера)

Публичные сертификаты в internal/shared/certs/assets/ (включаются в Docker-образ)

Важно: Никогда не добавляйте .certs_private в систему контроля версий.

2.3. Настройка SSH-агента и создание ключа пользователя
Клиент использует ssh-agent для выполнения криптографических подписей. Запустите агент и сгенерируйте ключ:

```bash
eval "$(ssh-agent -s)"
ssh-keygen -t ed25519 -N "" -f ~/.ssh/id_ed25519 -C "gophkeeper-user"
ssh-add ~/.ssh/id_ed25519
```

Убедитесь, что ключ загружен:

```bash
ssh-add -l
```

## Часть 3. Запуск серверной инфраструктуры через Makefile

### 3.1. Обзор Makefile

Проект содержит Makefile со следующими основными целями:

- **`make certs`** – генерирует PKI-сертификаты (Server CA и Device CA).
- **`make build-linux`** – собирает статические бинарники клиента и сервера для Linux.
- **`make up`** – последовательно выполняет `certs`, `build-linux`, а затем поднимает Docker-контейнеры с помощью `docker compose up -d --build`.
- **`make down`** – останавливает и удаляет контейнеры, но сохраняет данные в томах.
- **`make clean`** – полная очистка: останавливает контейнеры, удаляет локальные SQLite-базы клиента, сгенерированные protobuf-файлы и собранные бинарники.

### 3.2. Сборка и запуск контейнеров

Убедитесь, что вы находитесь в корневой папке репозитория, и выполните:

```bash
make up
```

Эта команда автоматически:
- Сгенерирует сертификаты (если они ещё не созданы).
- Соберёт бинарники для Linux.
- Запустит контейнеры PostgreSQL и gRPC-сервера в фоновом режиме.

Проверьте состояние контейнеров:

```bash
docker compose -f infra/compose.yml ps
```

Для просмотра логов используйте:

```bash
docker compose -f infra/compose.yml logs -f
```

### 3.3. Остановка и очистка контейнеров

Для остановки сервисов (контейнеры будут остановлены, но данные и бинарники сохранятся):

```bash
make down
```

Если требуется полная очистка (удаление контейнеров, томов базы данных, локальных клиентских хранилищ и артефактов сборки):

```bash
make clean
```

Эта команда выполняет `down`, удаляет все сгенерированные файлы (включая SQLite-базы клиента и Protobuf-код) и бинарники.

## Часть 4. Сборка клиента и регистрация устройства

### 4.1. Компиляция клиентского бинарника

Если вы ещё не выполнили `make up`, который также собирает бинарники, вы можете сделать это отдельно:

```bash
make build-linux
```

Либо скомпилируйте только клиент напрямую:

```bash
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -ldflags "-s -w" -o ./build/linux/gophkeeper ./cmd/gophkeeper/main.go
```

Бинарник появится в `build/linux/gophkeeper`.

### 4.2. Инициализация локального хранилища

```bash
./build/linux/gophkeeper init
```

### 4.3. Регистрация устройства на сервере

```bash
./build/linux/gophkeeper register --server localhost:443
```

При успешной регистрации клиент получит mTLS-сертификат.

## Часть 5. Работа с секретами, синхронизация и дополнительные команды

### 5.1. Создание локальных записей и регистрация устройства (правильный порядок)

1. **Инициализация локального хранилища**:

```bash
./build/linux/gophkeeper init
```

2. **Создание тестовой записи**:

```bash
./build/linux/gophkeeper create --name my_secret --type text --payload "Hello, secure world!"
```

3. **Просмотр списка записей**:

```bash
./build/linux/gophkeeper list
```

4. **Получение и расшифровка записи**:

```bash
./build/linux/gophkeeper get --name my_secret
```

5. **Регистрация устройства на сервере** (можно выполнить до или после создания записей, но рекомендуется сначала создать локальные секреты):

```bash
./build/linux/gophkeeper register --server localhost:443
```

6. **Синхронизация с облаком** (отправка локальных изменений на сервер и получение обновлений):

```bash
./build/linux/gophkeeper sync
```

### 5.2. Другие полезные команды

- **Список всех записей:** `./build/linux/gophkeeper list`
- **Удаление записи по ID:** `./build/linux/gophkeeper delete --id <UUID>`
- **Версия клиента:** `./build/linux/gophkeeper version`

### 5.3. Устранение типичных проблем

**Проблема:** Клиент не может подключиться к серверу (`connection refused`).
- Проверьте, что контейнер сервера запущен: `docker ps`.
- Убедитесь, что порт на хосте не занят другим процессом.
- Если вы изменили порт в compose-файле, укажите его при регистрации.

**Проблема:** Ошибка при генерации сертификатов (`make certs`).
- Убедитесь, что установлен `openssl`. В Ubuntu он входит в базовый набор, но может отсутствовать – установите: `sudo apt install openssl`.
- Проверьте права на запись в текущую директорию.

**Проблема:** Регистрация завершается с ошибкой `ssh-agent` недоступен.
- Убедитесь, что агент запущен и ключ добавлен: `ssh-add -l`.
- При необходимости перезапустите агент и добавьте ключ заново.

**Проблема:** Синхронизация не работает, сервер возвращает ошибку аутентификации.
- Проверьте, что клиент зарегистрирован (файл сертификата существует в локальном хранилище).
- Убедитесь, что сервер использует те же приватные ключи CA, что и при генерации сертификатов (пути должны совпадать с монтированными в контейнер).
